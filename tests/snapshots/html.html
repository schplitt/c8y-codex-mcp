This section outlines the design guidelines for building applications on the Cumulocity platform. These guidelines are essential for creating a consistent, high-quality, and user-friendly experience that aligns with the Cumulocity brand.

Foundations

The basic principles of our design system, including color, typography, grid, and motion.

Design tokens

The low-level variables used to build our components, which can be used to customize the look and feel of your applications.

Icons

Comprehensive icon library with search, usage patterns, and implementation guidelines for visual consistency.

Accessibility

Guidelines for building accessible applications that can be used by everyone.

Patterns

A collection of reusable solutions for common UI challenges, including data actions, user feedback, and responsive design.

Layout patterns

Guidance on the fundamental page structures used in applications, such as list layouts, detail views, and dashboards.

Styling guidelines

Best practices for styling components with CSS, Less, and design tokens.

CSS utilities

Ready-to-use utility classes for spacing, layout, typography, and common styling patterns.

Writing

Guidelines for writing clear, concise, and user-friendly interface text.

By following these guidelines, you will be able to:

  • Build applications that are visually consistent with the core Cumulocity applications.
  • Create a seamless and predictable user experience.
  • Leverage the full power of the Cumulocity Web SDK to build applications faster.

We recommend starting with Foundations to understand core principles, then exploring Design tokens as the building blocks. Next, learn about Icons and Accessibility requirements before diving into Patterns and Layout patterns. Finally, master the implementation details with Styling guidelines and CSS utilities.
